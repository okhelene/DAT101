1. [Why is object-oriented development so important in programming?]

    2. [object-oriented development is important because it helps to structure code in a way that is reusable, and easier to maintain. By encapsulating data and behavior within objects, developers can create more complex, reusable systems.]

  2. [In Task 1, we create a static object called "AccountType." Explain what this object is used for and how this type of object can help us in software development.]

    3. [the static object "AccountType" is used to define different types of accounts with their associated properties and behaviors. This type of object helps in software development by providing a clear structure for account types, making it easier to manage and extend the functionality related to different account types.]

  3. [Discuss why we use encapsulation in classes. Why do we want to "hide" code within our class? Explain how we can create "private" and "public" members in a constructor function.]

    4. [we use encapsulation in classes to protect the internal state of an object and to control access to its data and behavior. By hiding code within our class ("#"), we can prevent external code from directly and unintentionally modifying the object. In a constructor function, we can create "private" members by defining variables within the constructor function scope (using var, let, or const) that are not accessible outside the function. "Public" members can be created by attaching properties and methods to "this", making them accessible from outside the object.]

  4. Explain what happens when we use the "new" command in JavaScript.

    5. [when we use the "new" command, it creates a new instance of an object. This involves allocating memory for the new object, setting up the prototype chain, and calling the constructor function with the specified arguments. The "new" command allows us to create multiple instances of an object with the same structure and behavior defined by the constructor function.]
