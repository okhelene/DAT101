  1. [In this assignment, we use the DOM. Explain how this library can help us develop interactive web pages.]

    2. [DOM = document object model. its a programing interface built into browsers that represent your HTML page as a tree of objects. every element (like headings and buttons) becomes a node in that tree that javascript can reach through document. this makes it possible to interact by reading (get current value of an input), modify it (change text, hide element, add new element) and react to the user (respond to clicks, key presses, form submission). Without DOM you would only be able to deliver a static page]

  2. [What is an event? simply explain what happens when we add an event listener to an object]

    3. [an event is a signal that something happened, the user clicked a button, pressed a key, moved the mouse, or the browser finished loading resource. the browser fires these signals continously as the user interacts with the page when you call "element.addEventListener("click", myFunction), you are telling the browser: "whenever this element recieves a click, call "myFunction"". the browser keeps an internal list of these subscriptions. when the event fires, it works through that list and calls each registered function in order. nothing runs until the event actually happens, so the rest of your code is not blocked.]

  3. [sometimes we need to use an event object to get data from the event. explain how this event object is structured, dont explain all functions or attributes. try to explain, for example, how to retrieve data from a keyboard or mouse]

    4. [every time an event fires, the browser automatically creates an event object and passes it as the first argument to your handler. this object always has a common base (the type of event, the element that triggered it etc) and then it is extended with extra properties depending on what kind of event it is this you see in controllog eks: getting data from keyboard event: the event object will be "KeyboardEvent". the most useful properties are "event.key"(the human readable name of the key,e.g."Enter", "a") and "event.code"(the physical key position,e.g"KeyA"). Modifier state is available via "event.shiftKey", "event.ctrlKey" etc]
